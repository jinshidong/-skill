## ✅ 支持且常用的标签与属性

以下标签在公众号文章中常见且一般不会被剥离（但仍有可能会限制某些属性）：

* `<p>`：段落标签，用于正文、标题、段间分割。
* `<img>`：图片标签，用于插图。
* `<span>`：文本范围内的样式区分标签。
* `<strong>`：加粗标签。
* `<em>`：斜体或强调标签。
* `<br>`：换行标签。
* `<pre><code>`：用于代码块的组合标签（视编辑器版本而定，有的可能被改写）。
* `<ul>`, `<ol>`, `<li>`：无序／有序列表标签（在很多文章里可用）。
* `<blockquote>`：引用块标签，在某些编辑器里也可用。

**常见允许的样式（内联 style）属性**：

* `color`（文本颜色）
* `background-color`（背景颜色）
* `font-weight`
* `font-size`
* `text-align`
* `line-height`
* `font-family`（但在微信里可能被重置）
* `white-space`（部分版本可保留）
* `max-width`, `height:auto` 等图片适配样式

这些标签 + 属性组合，是目前主流公众号排版中“自己写 HTML”仍然可用的方式。

## 🚫 常被过滤／剥离的标签或属性

以下是编辑器中经常被清洗或不稳定支持的：

* `<div>`, `<section>`, `<header>`, `<footer>`, `<article>` 等结构性标签 — 编辑器往往不识别或会重写。
* `<style>`, `<link>`, `<script>` 等 CSS/JS 控制标签 — 微信会剥离以防安全风险。
* `class`, `id` 属性 — 编辑器常剥离这些，以保持统一渲染。
* 部分复杂 CSS 属性：如 `border-radius`, `box-shadow`, `position:absolute`, `float`, `margin` 等会被部分版本剥离或忽略。
* 外部 CSS 文件引用、@import 样式表。
* `<video>`、`<audio>`、部分 `<iframe>` 嵌入在公众号文章正文里常受限。比如 `<video>` 在微信浏览器中也可能强制全屏。 ([Nanmu][1])

## 🧪 实践建议

* 在 **电脑端编辑器**中进行排版测试，因为手机或 iPad 编辑器可能功能受限。
* 尽量使用上面 “可用标签” + “允许样式属性” 来写模板，避免使用被过滤标签。
* 对于你所关心的“圆角边框”“自定义背景色”等视觉效果，优先使用内联 `style` 的可兼属性（如 `border:1px solid`, `background-color`, `padding`, `border-radius`）并手动测试是否被编辑器剥离。
* 如果某些 CSS 效果被剥离，考虑用图片方式替代（例如将卡片背景或圆角框做成图片）。
* 在粘贴 HTML 时避免落入“插入代码块”模式，因为那会把 HTML 显示为纯文本。要粘到正文视图中。
* 经常保存稿件为草稿、预览在手机端查看效果，因为编辑器版本可能因账号、类型（订阅号/服务号）或微信后台改版而不同。
