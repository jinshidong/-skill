自动微分中的连续性问题

日期：2020-05-22　标签：自动微分 / 选择语句 / 连续性问题 / 人工智能

如何解决自动微分中选择语句的连续性求导问题
在理科及工科中，总会遇到数值计算。面对观测方程，进行数据拟合、线性化、积分、非线性优化与梯度下降等，这些过程我们需要自动微分。它在非线性优化、扩展卡尔曼滤波、变分同化及机器学习中有极大的用途。

一、微分方法
1）解析式求导 最常用也最准确，但手动计算复杂繁重。例：sin'(x)=cos(x)，cos'(x)=-sin(x)。当模型含多参数（如球谐函数）时，人工求偏导工作量很大。一般可用链式法则：∂f/∂vᵢ = Σⱼ( ∂f/∂uⱼ · ∂uⱼ/∂vᵢ )。

2）差分近似 前向差分：[f(x+h)-f(x)]/h；后向差分：[f(x)-f(x-h)]/h；中心差分：[f(x+h/2)-f(x-h/2)]/h。步长 h 选择困难：过大精度差，过小截断与数值误差突出。

3）链式自动微分 计算函数值的同时得到各参数导数，避免差分对 h 的依赖。但模型中的条件/分段/循环会带来不连续，使导数异常，影响反向传播或梯度下降。

二、用平滑选择函数缓解不连续
ReLU 在阈值处导数不连续。可用 Sigmoid 平滑函数 S(x,a,u)=1/(1+exp(-(x-a)/u))（u 控制过渡宽度）把“硬切换”变“软过渡”，用于分段函数拼接。


线性两段在阈值 a₀ 附近用 S 平滑拼接，可得连续且可导的近似；当两段非线性差异较大时，衔接区误差可能增大，需根据精度选择 u。



三、代码示例

double f(double a)
  {
    static double a0 = 5;
    static double k1 = 0.3;
    static double k2 = 0.6;
    return a > a0 ? k1*(a-a0) : k2*(a-a0);
  }


// autodiff_tnna.h (节选)
template<class Cell>
struct autodiff{
  Cell _val;
  std::valarray<Cell> _dval;
  autodiff(const Cell& val = Cell(), std::valarray<Cell> dval = {}) : _val(val), _dval(dval){}
  autodiff<Cell> operator*=(const autodiff<Cell>& a){
    auto val=_val, dval=_dval;
    _val = val*a._val;
    _dval = dval*a._val + val*a._dval;
    return *this;
  }
  // ...
};


结论
在存在条件/分段的不连续场景中，使用 Sigmoid 等平滑函数将“硬切换”变为“软过渡”，配合链式自动微分可获得稳定梯度；过渡参数 u 需结合精度与数值稳定性调节。
